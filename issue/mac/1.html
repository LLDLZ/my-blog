    关于“zsh: permission denied”的解决方法

    # 关于“zsh: permission denied”的解决方法

    # 背景

    使用自己搭建的脚手架命令新建文件和文件夹一直执行错误

    # 环境 macbook

    # 查找 npm 脚本目录

    // 查找目录npm bin目录
npm bin -g
cd /usr/local/bin
// 找到命令所在的路径
chmod u+x 你的命令

    # 命令说明

    chmod是权限管理命令change the permissions mode of a file的缩写。\
u代表所有者。x代表执行权限。’+’ 表示增加权限。\
chmod u+x file.js 就表示对当前目录下的file.js文件的所有者增加可执行权限。
